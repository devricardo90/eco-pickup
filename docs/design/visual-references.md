# EcoPickup - Visual References And Direction

## Purpose

This document defines the visual direction for EcoPickup without copying any specific brand. It should guide design exploration and future UI polish work.

## Visual Principles

### Sustainable

Sustainability should appear through calm tone, responsible service language, and subtle color choices. Avoid literal eco decoration as the primary design device.

### Clean

The interface should be easy to scan. Forms, lists, status panels, and timelines should have clear structure and generous but controlled spacing.

### Trustworthy

The product handles pickup requests, addresses, scheduling, and operational states. It should feel reliable and grounded.

### Modern

Use contemporary UI composition, but avoid trendy decoration that will age quickly or distract from the workflow.

### Simple

The user should understand the flow quickly: request, review, schedule, track. Screens should not require explanation.

### Professional

The portfolio presentation should show product maturity. It should not look like a default scaffold or tutorial project.

### Accessible

Contrast, focus, labels, states, and mobile layouts must be practical and readable.

## Avoid

- generic template look
- excessive green
- childish eco visuals
- heavy corporate look
- decorative illustrations without function
- startup fake style
- gradient-heavy decoration with no product meaning
- purely marketing composition that hides the actual app workflow
- icons or visuals that imply unsupported features

## Suggested Reference Categories

Use these as categories, not brands to copy.

### Clean SaaS Dashboard

Look for:

- clear page headers
- structured cards
- readable tables/lists
- restrained action hierarchy
- consistent badges and statuses

Use for:

- owner dashboard
- request list
- request metadata

### Civic / Service Request Apps

Look for:

- practical service language
- clear forms
- address/request details
- user confidence around submitted requests
- accessible public-service tone

Use for:

- landing page
- create pickup request
- request confirmation states

### Logistics / Status Tracking UI

Look for:

- timeline patterns
- status badges
- step progression
- clear timestamps
- exception/error states

Use for:

- request detail
- tracking timeline
- lifecycle cards

### Sustainability Service Landing Pages

Look for:

- restrained sustainable identity
- service explanation
- trustworthy impact language
- real workflow emphasis

Use for:

- public landing
- value proposition
- educational copy

## Notes For Designer

Create a system, not just prettier screens.

Prioritize:

- hierarchy
- spacing
- language
- components
- states
- responsive behavior

The design should make the validated demo flow feel intentional from start to finish. A visitor should be able to understand the product, sign in, create a request, and track it without seeing internal delivery language or inconsistent UI patterns.

Do not overfit to screenshots. The output should be reusable by frontend implementation tasks and resilient as the product grows.
